Security & Privacy Analysis

Mobile App Security Framework

Unit Learning Outcome 4 (ULO4) Compliance Report

1. Secure Data Transmission

This application enforces HTTPS (TLS/SSL) for all RESTful API calls to prog2005.it.scu.edu.au. This prevents Man-in-the-Middle (MITM) attacks and ensures data integrity during transit.

2. Critical Asset Protection

Specific business logic prevents the deletion of high-value assets. For example, the "Laptop" record is hard-coded as a protected item in the frontend controller to prevent accidental or malicious removal.

3. XSS & Injection Prevention

By leveraging Angular’s built-in DOM Sanitizer, the app automatically encodes user-provided strings. This mitigates Cross-Site Scripting (XSS) risks when rendering inventory notes or names.

4. Strict Type Validation

Using TypeScript Interfaces, the app enforces strict data structures. Numeric fields like Price and Quantity are validated before being sent to the POST/PUT endpoints to ensure database consistency.

5. Data Minimization

The app follows a Stateless Privacy Model. No sensitive inventory or supplier data is stored in LocalStorage or IndexedDB, reducing the risk of data theft from a lost or stolen device.

6. Feedback & Errors

Security through transparency: The app uses Toast Notifications to provide immediate feedback on API failures without exposing internal system stack traces to the end-user.
